Intro

Graph Coloring Problem is NP-Complete, thus, we believe Graph Coloring Problem is a great way of testing the performance of different architectures due to its nature of running complex algorithm to iterate through constraints and find a solution. The more complicated a map, meaning the more nodes of a map it has, and the more constraints there are among each node, the more steps will be added to the algorithms to run through. Thus, comparing the algorithm's running time among different machines will get a hint of what are the factors that could affect an architecture's performance, such as CPU, memory, or GPU.

In this project, we have provided these 6 pre-built maps for testing

  • Map of Australia
  • Map of USA
  • 100 Nodes, previously generated
  • 125 Nodes, previously generated
  • 130 Nodes, previously generated
  • 135 Nodes, previously generated

(Why we made only 5 nodes of incrementation for the map? Since as the number of nodes increase, technically the difficulty of the map increases exponentially, therefore the running time would increase exponentially.)

And, we utilize these 7 algorithms to color these map

  • Backtracking
  • Dsatur
  • Lmxrlf
  • Mcs
  • TabuCol
  • Hybrid Lmxrlf
  • Hybrid Dsatur

As previously mentioned, the algorithm code are borrowed from these two repositories(thanks!) -

  • https://github.com/brrcrites/graph-coloring
  • https://github.com/chenxuhao/csrcolor

We have also provided a way to let you randomly generate a map of the number of nodes at your choice and test the performance of your machine. If you would like, you can also copy this map file to the other machines, run and compare. More detailed info will be explained in the below sections.
